# PATIENT.md – Patient Portal (Self‑Report Intake Form with Status Tracking) ## Overview The patient portal allows individuals to **complete a self‑reported health intake form** based on the Comprehensive Nursing Assessment document. After submission: - The patient receives a confirmation **email and/or SMS**. - The request appears in the admin/nursing section for an RN to review. - The patient can **track the status** of their request (Pending, In Review, Completed) and receive updates when the RN completes the assessment.>**Patient cannot see or edit the RN’s clinical findings.** Only the intake data and status. --- ## Page 1: Patient Dashboard **URL:** `/portal/dashboard` After login,
the patient sees: - Welcome banner - Upcoming appointments - Outstanding balance - **Active requests card** – shows pending nursing assessment requests with status - **Quick action:** “Start a Nursing Assessment Intake” (button) - **Link to request history:** “View all my requests” --- ## Page 2: Pre‑Nursing Assessment Intake Form **URL:** `/portal/pre-nursing-intake` ### Purpose The patient fills out all sections that do not require clinical judgment. Data can be saved as a draft and submitted. ### Form Sections (All Patient‑Entered) *(Same as previously documented – see original PATIENT.md for full field list.)* ### Notification Preferences on Submission Before submitting,
the patient selects how they want to receive updates: | Option | Description | |--------|-------------| | **Email** | Send updates to my registered email address (pre‑checked) | | **SMS** | Send text messages to my mobile phone (requires verified phone number) | | **Both** | Email and SMS | These preferences are saved for future requests. ### Form Actions | Button | Behavior | |--------|----------| | **Save Draft** | Saves all entered data without notifying clinic. Patient can return later. | | **Submit to Clinic** | Final submission. After confirmation: <br>- Data becomes read‑only for patient <br>- A **new request** is created in the admin/nursing section with status “Pending” <br>- Confirmation **email** is sent (subject: “Your nursing assessment intake has been received”) <br>- If SMS opted in,
a text message is sent: “Your intake was received. We will notify you when it is reviewed.” <br>- Patient is redirected to the **Request Status** page | | **Cancel** | Discard changes (with warning if unsaved). | --- ## Page 3: Request Status & Tracking **URL:** `/portal/requests` This page lists all intake requests submitted by the patient,
with real‑time status updates. ### Status Definitions | Status | Meaning | What patient sees | |--------|---------|-------------------| | **Pending** | Submitted,
not yet reviewed by RN | “Your request is waiting for a nurse to review it.” | | **In Review** | RN has started working on the assessment | “A nurse is reviewing your information and may contact you if needed.” | | **Completed** | RN has finished and signed the nursing assessment | “Your assessment is complete. You can view a summary below.” | | **Action Needed** | RN requires more information from patient | “Please provide additional information (see message).” | ### Request Table | Column | Description | |--------|-------------| | Request Date | Date submitted | | Status | Badge (Pending / In Review / Completed / Action Needed) | | Last Update | Date of last status change | | Actions | View Summary button | ### Request Detail View Clicking a request opens a read‑only summary of: - **What you submitted** (all patient‑entered data) - **Status timeline** (submitted → in review → completed, with timestamps) - **Nurse’s message** (if any, e.g., “Please call the clinic to clarify your medication list.”) - **Completed assessment summary** (only if status=Completed) – shows limited clinical summary: nursing diagnoses,
goals,
medication delegation instructions,
next review date. **No raw clinical data** (e.g., vital signs, AIMS score) unless configured. ### Notifications on Status Change When the RN updates the request status (e.g., from Pending to In Review, or to Completed),
the patient automatically receives: - **Email** (full details + link to portal) - **SMS** (short notification: “Your nursing assessment request has been updated. Log in to view.”) The patient can unsubscribe from SMS/email notifications in Profile settings,
but will still see updates when logging in. --- ## Page 4: Profile & Notification Settings **URL:** `/portal/profile` Add a new section: ### Notification Preferences for Requests | Setting | Options | |---------|---------| | Email notifications | On/Off (default On) | | SMS notifications | On/Off (requires verified phone) | | Receive updates when RN completes assessment | On/Off | These settings are independent of appointment reminders. --- ## Integration with Admin / Nursing Section When a patient submits the intake form: 1. A new record is created in `nursing_assessment_requests` with: - Patient ID - Submitted data (JSON) - Status: `pending` - Notification preferences (email, SMS) 2. The RN sees this in **Nursing Assessments** list (under Clinical sidebar) with an **“Import”** button. 3. When the RN clicks **Import**,
a draft nursing assessment is created,
pre‑filled with patient data. The request status changes to `in_review`. 4. When the RN **signs** the assessment,
the request status changes to `completed`. The system automatically: - Sends email/SMS to patient (based on their preferences) - Updates the patient’s request detail view with a summary of the completed assessment. 5. If the RN needs more information,
they can add a comment and change status to `action_needed`,
triggering a notification to the patient to log in and respond. --- ## Technical Notes for Developers - Use a background job queue to send emails/SMS (so submission is fast). - SMS gateway (e.g., Twilio) must be configured in Admin → Integrations. - Email templates: customizable by admin (Admin → Settings → Notifications). - Status changes are logged in audit logs. - Patient can view but never edit submitted data after submission. --- *This patient portal now provides full visibility: submit → receive confirmation → track progress → get notified on completion.*
